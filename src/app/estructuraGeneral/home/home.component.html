Descripción:
Crear aplicación web para la gestión de usuarios. La información a almacenar es la siguiente:
Nombre. Apellido Paterno. Apellido Materno. Dirección. E-mail. Telefóno (Puede tener más de uno). Departamentos (Puede tener más de uno. Ej. Depto1, Depto2, etc) La aplicación deberá cargar información masivamente y en línea. El resto de operaciones serán solamente sobre la aplicación. Aplicación CRUD. Despliegue de usuarios en el sistema.

Criterios de aceptación:
Success Response (Response Entity "timestamp") General Response Error (Response Entity "Mensaje error + timestamp" (HTTPStatus)) Logging entrance and response for every entry point using AOP. Name convenctions (/api/version "v1"/dominio "users"/operation"add") Methods to call MS from external parts (POST, GET) Define HTTPStatus code, how to use each one and in which moment.

  • 200 = success
  • 400 = bad request
  • 404 = path not found / data not found
  • 500 = exception

Implement Global Handler Exception to management the errors (RestControllerAdvice @ExceptionHandler) //Identify what artifact repository we should use (GitLab) Define a standard to formatting of source code.
